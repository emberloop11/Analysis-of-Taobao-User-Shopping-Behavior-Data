# Analysis-of-Taobao-User-Shopping-Behavior-Data

## 一、项目背景

选取 200 万条数据，通过 MySQL 对数据进行预处理，去除空值、重复值、异常值，分别从用户和商品两个角度进行分析，计算用户流量、留存率、热门商品及品类，对用户购买行为进行漏斗分析，使用 RFM 模型对用户进行定位，计算热门品类和热门商品并对商品特征进行分析。然后通过 Tableau 软件实现数据可视化并搭建仪表盘。最后从平台新老客占比、浏览高峰期、短期留存率、转化率、热门商品、热门品类等方面给进行总结并给出建议。

## 四、分析流程与核心指标

整个分析分为以下四个步骤，覆盖了电商分析中常用的核心指标：

1. **数据导入与清洗（MySQL）**  
   处理缺失值、时间戳异常、重复记录，构建用户行为宽表。

2. **统计分析（SQL）**  
   - 日活（DAU）、周活（WAU）  
   - 各类行为（点击、收藏、加购、购买）的频次分布  
   - 漏斗分析：点击 → 收藏/加购 → 购买  
   - 用户复购率（基于购买次数 ≥ 2 的用户占比）

3. **探索性可视化（Python）**  
   - 用户行为分布柱状图  
   - 用户活跃度直方图  
   - 按小时的行为热力图（发现活跃时段）

4. **交互式仪表板（Tableau）**  
   - 核心 KPI 看板  
   - 时间、类目、行为类型筛选  
   - 转化漏斗图、复购率趋势、Top 商品类目

## 五、关键分析结论（示例）

根据本次分析发现的主要问题和机会点如下：

- **转化瓶颈**：从“加购”到“购买”的转化率明显偏低，建议优化购物车页面并增加限时优惠提示。
- **用户结构**：约 15% 的高活跃用户贡献了 65% 的购买行为，可设计会员等级或专属权益以提升留存。
- **商品运营**：3 个头部类目贡献了超过一半的收藏与加购行为，建议给予流量倾斜并搭配组合推荐。
- **时间规律**：晚间 20:00–22:00 为购买最高峰，适合在此时间段内推送促销活动或直播带货。

> 注：以上结论基于当前数据分析得出，具体数值可根据实际数据集重新运行 SQL 或 Python 脚本获得。

## 六、如何运行本项目

### 1. 准备数据
从百度网盘下载 `UserBehavior.csv`，导入 MySQL（建议使用 `LOAD DATA INFILE` 或分批导入）。

### 2. 执行 SQL 分析
在 MySQL 中运行 `数据统计分析SQL代码.sql` 中的查询，可将结果导出为 CSV 供后续可视化使用。

### 3. 运行 Python 探索性分析
使用 Jupyter Notebook 打开 `数据清洗部分可视化结果.ipynb`，按顺序执行单元格（需提前安装 Pandas、Matplotlib）。

### 4. 查看 Tableau 仪表板
用 Tableau Desktop 打开 `可视化分析.twb`，重新连接到你的数据源（更新数据连接路径）。
